# Architecture Decision Records

ADRs capture the context, options considered, and reasoning behind significant architectural decisions. Once accepted, they serve as the authoritative record of *why* the system is built the way it is.

## Status definitions

| Status | Meaning |
|---|---|
| **Proposed** | Under discussion, not yet decided |
| **Accepted** | Decision made and in effect |
| **Deprecated** | Was accepted but no longer applies |
| **Superseded** | Replaced by a later ADR |
